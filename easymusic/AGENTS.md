# 项目协作约定

## 语言

- 所有思考过程必须使用中文
- 所有回复必须使用中文

## 项目定位

当前项目是一个用于毕业设计的 AIGC 音乐平台二次开发项目，目标不是简单运行现有源码，而是在现有基础上做出真实、完整、可答辩的个人改造成果。

当前统一主线为：

**用户一句话描述需求 -> AI 扩写成结构化提示词 -> 生成音乐 -> AI 生成封面 -> 积分结算 -> 作品发布与统计**

## 会话启动必做

每次在当前目录打开新窗口并开始工作前，必须优先读取以下文件：

1. `project_memory/00-项目总目标.md`
2. `project_memory/01-当前状态.md`
3. `project_memory/06-待办清单.md`
4. `project_memory/07-本轮任务.md`
5. `毕业设计二次开发实施规划.md`

如果任务与排错、历史决策、需求变更有关，还需要继续读取：

1. `project_memory/02-需求变更记录.md`
2. `project_memory/03-问题排查记录.md`
3. `project_memory/05-实现决策.md`

## 工作规则

- 开始编码前，先根据上述文件判断当前项目正在做什么、做到哪一步、下一步应该完成什么
- 如果发现用户需求已经变化，先更新 `project_memory/02-需求变更记录.md`，再改代码
- 如果发现当前状态已过期，先更新 `project_memory/01-当前状态.md`
- 如果当前正在推进一个明确任务，优先维护 `project_memory/07-本轮任务.md`，记录当前目标、进展、阻塞项和下一步
- 每完成一个阶段性改动，更新 `project_memory/04-开发日志.md`
- 每修复一个明确问题或 bug，更新 `project_memory/03-问题排查记录.md`
- 每做出重要设计取舍，更新 `project_memory/05-实现决策.md`
- 每次结束工作前，必须更新 `project_memory/06-待办清单.md`
- 每次结束工作前，如果本轮任务未完成，也必须更新 `project_memory/07-本轮任务.md`

## 安全规则

- 不要把真实密钥直接写入日志或规划文档
- 外部 AI 的 URL 和 Key 统一来自项目根目录下的 `api的url和key.txt`
- 不要在日志中打印完整 Key

## 记录目标

`project_memory` 的用途不是堆零散内容，而是为了：

- 在重开窗口后快速恢复上下文
- 追溯需求变化、设计决策和排错过程
- 为论文、答辩、周报提供可复用材料

如果没有明确相反要求，应持续维护这些记录文件。
