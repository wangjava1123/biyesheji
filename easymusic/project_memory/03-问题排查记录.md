## 2026-04-28 问题 003

- 问题现象：前台列表页存在滚动判断不稳定，部分页面在移动端或内容不足一屏时容易出现不继续加载、滚动口径混乱的问题
- 复现条件：
  1. 打开依赖 `DataLoadMoreList` 的作品列表页
  2. 列表容器本身可滚动，但组件同时监听了容器滚动和 `window` 滚动
  3. 当首屏数据不足以撑满容器，或移动端 viewport 高度变化时，更容易出现加载判断异常
- 原因分析：
  1. `DataLoadMoreList` 内部额外绑定了 `window` 的 `scroll` 事件，和组件自身滚动容器职责重叠
  2. 组件没有在“内容未撑满容器”的情况下主动补拉下一页
  3. 创作者中心、创作页等区域又依赖内层滚动容器，因此更需要统一在组件内部收口滚动判断
- 处理方案：
  1. 移除 `DataLoadMoreList` 对 `window` 滚动的额外监听
  2. 保留组件容器内滚动触发加载的方式
  3. 增加“内容不足一屏时自动补拉下一页”的自检逻辑
- 影响文件：
  - `easymusic-front/easymusic-front-web/src/component/common/DataLoadMoreList.vue`
  - `easymusic-front/easymusic-front-web/src/views/idea/Idea.vue`
  - `easymusic-front/easymusic-front-web/src/views/my/My.vue`
- 最终结果：已修复

- 处理方案：
  1. 在 `MusicCreteApi4TianpuyueImpl` 中增加创建前配置校验
  2. 对天普乐接口返回的 `status / message` 做显式解析

## 2026-04-28 问题 008

- 问题现象：创作页“智能扩写”回填出来的音乐提示词和封面提示词过于简单，缺少专业制作信息，且音乐与封面之间没有稳定闭环
- 复现条件：
  1. 在创作页输入一句话需求并点击“智能扩写”
  2. 查看 `musicPrompt / imagePrompt`
  3. 当远程对话 AI 超时后，系统会走本地 fallback，结果通常只有“风格 / 情绪 / 人声”级别的弱模板
- 原因分析：
  1. `PromptGenerateApiOpenAiImpl` 旧版 system prompt 过弱，只要求返回几个 JSON 字段，没有约束专业度
  2. 远程请求超时后，本地 fallback 只有简单的 genre / emotion / voice 拼接
  3. 扩写接口只接收 `rawPrompt`，没有利用当前“歌曲 / 纯音乐、模式、曲风、情绪、人声”等上下文
  4. 前端回填后也没有强制同步曲风 / 情绪 / 人声，导致提示词与表单设置可能分离
- 处理方案：
  1. 为扩写接口新增上下文参数：`musicType / modeType / musicGenre / musicEmotion / musicSex`
  2. 强化对话 AI system prompt，明确要求输出专业级音乐 brief 与闭环封面概念
  3. 重写本地后处理与 fallback 逻辑，无论远程是否成功，都统一生成更专业的音乐提示词和封面提示词
  4. 前端扩写回填时同步更新曲风 / 情绪 / 人声，减少表单与提示词脱节
- 影响文件：
  - `easymusic-java/easymusic-common/src/main/java/com/easymusic/entity/dto/PromptAssistContextDTO.java`

## 2026-04-28 问题 009

## 2026-04-28 问题 012

- 问题现象：点击创作页“创作音乐”第一次直接提示“请求超时，请稍后重试”，但刷新“我的作品”后能看到新作品已经是“生成中”
- 复现条件：
  1. 登录前台并进入创作页
  2. 填写提示词后点击 `创作音乐`
  3. 请求超过 10 秒时，页面先报超时；随后刷新作品列表，可见新记录已经写入且状态为 `生成中`
- 原因分析：
  1. 前端 `Request.js` 的全局 axios 默认超时仍是 10 秒
  2. `Create.vue` 中 `createMusic` 请求没有像“智能扩写”一样单独放宽超时时间
  3. 后端 `/my/createMusic` 会先完成积分扣减、创作记录落库和作品 `CREATING` 状态写入，再返回响应
  4. 因此前端会先因 10 秒超时中断，但后端真实创建流程已经启动，形成“前端假失败、后台已生成中”
