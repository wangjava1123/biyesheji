## 2026-04-27 问题 001

- 问题现象：点赞功能存在记录，但列表和统计所依赖的点赞数字段可能不准确
- 复现条件：用户执行点赞/取消点赞后，`music_info_action` 会变化，但 `music_info.good_count` 未同步维护
- 原因分析：当前点赞实现只维护行为明细表，没有同步更新聚合字段
- 处理方案：
  1. 在 `MusicInfoMapper` 中新增 `updateMusicGoodCount`
  2. 在 `MusicInfoActionServiceImpl#doGood` 中，点赞和取消点赞时同步增减 `good_count`
  3. 在 `stage1_upgrade.sql` 中增加历史点赞数回填逻辑
- 影响文件：
  - `easymusic-java/easymusic-common/src/main/java/com/easymusic/service/impl/MusicInfoActionServiceImpl.java`
  - `easymusic-java/easymusic-common/src/main/java/com/easymusic/mappers/MusicInfoMapper.java`
  - `easymusic-java/easymusic-common/src/main/resources/com/easymusic/mappers/MusicInfoMapper.xml`
  - `easymusic-java/stage1_upgrade.sql`
- 最终结果：已修复

## 2026-04-27 问题 002

- 问题现象：作品切回草稿时，`publish_time` 可能保留旧值
- 复现条件：作品先发布，再切换为草稿
- 原因分析：原有通用更新 SQL 只会更新非空字段，`publishTime = null` 不会真正落库
- 处理方案：
  1. 在 `MusicInfoMapper` 中新增 `updateMusicPublishInfo`
  2. 使用显式 SQL 直接更新 `publish_status` 与 `publish_time`
  3. 在服务层根据目标状态决定发布时间是当前时间、保留原值还是置空
- 影响文件：
  - `easymusic-java/easymusic-common/src/main/java/com/easymusic/service/impl/MusicInfoServiceImpl.java`
  - `easymusic-java/easymusic-common/src/main/java/com/easymusic/mappers/MusicInfoMapper.java`
  - `easymusic-java/easymusic-common/src/main/resources/com/easymusic/mappers/MusicInfoMapper.xml`
- 最终结果：已修复
